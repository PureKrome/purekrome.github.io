# ANNOUNCEMENT: Code Coverage Showdown is now available!

I've created a new website page which compares Microsoft's Code Coverage tool with Coverlet for .NET projects. The page includes a detailed comparison of the two tools with respect to line, branch, and method coverage.

![Dashboard](@/assets/images/code-coverage-showdown-dashboard.png)
_Image credit: Justin Adler_

# Why did I create this page?

I've been using Coverlet since I first learnt about code coverage in .NET and the important value it brings to testing and quality. At the end of 2025, Microsoft updated their testing tool called Microsoft Testing Platform (MTP) to replace the aging VSTest harness. One of the featuers of MTP was a code coverage tool - Microsoft Code Coverage. _At the same time_ Coverlet was not updated to work with MTP, so I decided to switch over to Microsoft Code Coverage to get some results.

Just a few days ago, Coverlet was updated to work with MTP so I thought it would be interesting to compare the two tools and see how they stack up against each other.

# Dynamically updating results

The page was designed to run every night and check if there's a newer version of

- Microsoft Code Coverage
- Coverlet

If one of these is updated, then we run the tests and update the results on the page.

# Run tests ... for what code?

For no particular reason, I chose to run the tests on the [Humanizer](https://github.com/Humanizr/Humanizer) library, a popular .NET library that provides human-friendly string manipulation and formatting.
