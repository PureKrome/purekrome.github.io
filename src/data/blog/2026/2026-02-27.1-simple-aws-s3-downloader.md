# Simple: AWS S3 Downloader

🤖 GitHub: <a href="https://github.com/PureKrome/SimpleAWS.S3.Downloader">Simple: AWS S3 Downloader</a>

I made a simple TUI (Text UI) console application for downloading files from AWS S3. It's built using .NET and C#. The idea was to create a simple tool that allows you to quickly download files from S3 without having to use the AWS CLI or write a bunch of code.

I understand that every developer will scream "AWS CLI + scripting is better!" and that's probably true for many. For me, I wanted a simple tool that didn't mean I needed to have the AWS CLI installed (shock horrow - some people don't want to install the AWS CLI) and I wanted it to be a TUI because I find them fun and nostalgic.

_I always wanted to use this experience to explore using AI to build a TUI Application_.

# Building a TUI Application with AI

I don't code FE's (Front Ends). I don't do design. I don't do UX. That's a skill which I admire to designers and FE's.

But with AI, I can now access simple FE skills to build a simple UI. (e.g. <a href="/posts/simple-observability/">Simple: Observability</a>)

The UI is very basic, but it gets the job done. You can enter your AWS credentials, select a bucket, and then select a file to download. It's not pretty, but it works.

# Reflections about using AI: It's me, not you

I found the experience exciting at first, then became frustrating. Early velocity, later challenges. Part of this experiment was to literally create a TUI. More so, I've _never_ made an interactive TUI! Early prompts and output was promising until I got to the point of making it interactive.

### Example 1: Single page TUI.

I wanted to have a 'single page' (similar to the old <a href="https://midnight-commander.org/">Midnight Commander</a> app) instead of text scrolling down the screen. This ment we needed to change from <a href="Spectre.Console">https://github.com/spectreconsole/spectre.console</a> to <a href="https://github.com/gui-cs/Terminal.Gui">Terminal.Gui</a>.

### Example 2: Visual Interactivity

Next I wanted to have the TUI get updated as we download items - instant visual feedback. This meant that callbacks and async code was needed to update the UI as the download progressed. This was a bit of a challenge for the AI to get right - constantly either not updating or exceptions were getting thrown. It took a bit of prompting and coaxing to get it working. What I hope was important here was that my _experience_ with using apps and what apps _do_ was an important "north star" for how I tried to keep directing the agent to finally get it working.

# The problem with options is that we have too many?!

I'm often asked "which model did you use?".

I used GPT 5.2 Codex via Copilot in Visual Studio.

I wanted to try Claude Opus 4.6 but it was "3x" the token "cost" and I'm on the entry level GitHub Copilot plan which has a token limit. GPT 5.3 Codex wasn't available at the time _to me_.

# Rough Edges v1 MVP

The app is very basic and has some rough edges.

But it works!

I'm sure there's a ton of TUI and GUI apps (that are free) that do this better, has heaps of bells and whistles and is more robust. But this was a fun experiment to see if I could build a TUI app with AI and it was a fun way to explore the challenges of building an interactive TUI app.
