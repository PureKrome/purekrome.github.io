# IISExpress Port Conflicts - How to Resolve Them

## The problem with legacy development in 2026
In 2026 no one should be using IISExpress for development (NEVER, EVA!) .... unless ... you're stuck maintaining legacy ASP.NET applications. If you are, then you may have come across the dreaded "IISExpress Port Conflict" error.

I've just had to open and debug an old .NET Full Framework ASP.NET Web Forms application and I was greeted with this error. I've been dreading having to have IISExpress install, let alone run on my machine ... but I had no choice.

The error message was _visually_ something I've not seen before - and I've done _heaps_ of ASP.NET development with FF and IISExpress over the years. So much so, I'm still in therapy from the trauma of "the bad old days".

The error messages are (in order, on Windows 11):

![Windows Notification error 1](@/assets/images/iisexpress-error-1.png)
![Windows Notification error 2](@/assets/images/iisexpress-error-2.png)
![IISExpress dialog error](@/assets/images/iisexpress-error-3.png)

and the log file info:

```
Failed to register URL "http://localhost:51110/" for site "Your-Old-WebApp" application "/". Error description: The process cannot access the file because it is being used by another process. (0x80070020)
```

Scary stuff!

At first, it's obvious because port 51110 is being used by another process. But .. I actually couldn't find any process using that port. I tried all the usual things - netstat, TCPView, etc. Nothing was using that port.

## 2026 says Hi 👋🏻

In 2026, we have Docker and WSL. These tools are great for development, but they can also cause port conflicts. More importantly, these tools can _reserve ports_ for their own use, which can cause conflicts with IISExpress.

In my case I typed this to see what ports were reserved:

```
netsh interface ipv4 show excludedportrange protocol=tcp
```
```
Protocol tcp Port Exclusion Ranges

Start Port    End Port
----------    --------
     44389       44389
     44399       44399
     49694       49694
     49709       49808
     49809       49908
     50000       50059     *
     50060       50159
     50160       50259
     50360       50459
     50560       50659
     50782       50881
     51016       51115   <-- 😱 This is the problem!
     52291       52390
     .... the rest are snipped for brevity ....

* - Administered port exclusions.
```

Blamo! There it is - port 51110 is in the range of ports that are reserved by Docker and WSL. This means that IISExpress cannot use that port because it's reserved for Docker and WSL.
